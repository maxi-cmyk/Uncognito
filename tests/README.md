# Tests

Cross-application verification lives here.

- `e2e` - browser-level demo flows across extension, web, and API surfaces.
- `fixtures` - reusable screenshots, payloads, and mock provider responses.

Demo coverage should include both manual roast creation and the Screenshot + LinkedIn Link path that returns a LinkedIn share URL without depending on LinkedIn API credentials.

Package-specific tests should live next to the package or app they verify.
